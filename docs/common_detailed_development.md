# 공통 상세 개발

상세 개발은 이미 있는 방식으로 기능을 조합하는 일이 아니라, 아직 답이 없는 기반의 질문을 푸는 일이다.

목적은 위에서 잡되, 답은 아래에서 찾는다. 조사하고, 작은 구현과 테스트로 확인하고, 그 결과로 인터페이스와 설계를 다듬는다. 이 과정의 중간 산출물은 다음 판단의 근거가 된다.

```text
방향
  → 탐색과 검증
  → 기반과 경계의 정리
  → 서비스 조합
  → 프로젝트에 녹이기
```

AI는 이 과정에서 작은 업무를 수행하고 결과를 만들 수 있다. 하지만 방향, 우선순위, 다음 작업의 조합은 사람이 계속 판단한다. AI의 제안은 결론이 아니라 다음에 확인할 재료다.

기반이 검증되고 재사용 가능한 경계가 생기면, 그 뒤의 일은 일반 개발로 넘어간다. 일반 개발은 축적된 지식과 구성 요소를 목적에 맞게 탑다운으로 조합한다.

다만 일반 개발 중 기존 기반으로 설명되지 않는 질문이 다시 생기면, 그 부분은 상세 개발로 돌아간다.

이 방식은 고정된 절차가 아니다. 실제 작업에서 필요한 만큼만 밟고, 반복되어 안정된 부분만 나중에 스킬이나 메모리로 남긴다.
